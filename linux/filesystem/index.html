Filesystem

 

Information about the Filesystem

How does the Filesystem work?
It starts at “/“. Every Folder is another layer in the Filesystem

example:
/home/user

Mounting a Drive

You just simply mount a Drive in a path of the System. This path can be anywhere but it’s recommended to mount at /mnt.

example:
/mnt/drive1

Drive naming scheme in Linux

sda – 1st Drive
sdb – 2nd Drive
sda1 – 1st Drive/1st Parition
sdc4 – 3nd Drive/4th Parition


The most popular file systems in Linux

ext4

Ext4 was introduced in 2008 as a successor to ext3. The file system is currently standard on many Linux systems like Ubuntu. The most-fac-reaching innovation was the extents feature which optimizes the management of large files and prevents fragmentation more efficiently than its predecessors. Under ext4, partitions can be enlarged and reduced also during the operation. The maximum file system size was limited to 32 TB with ext3. The limit with ext4 is one exabyte ( one million TB)

exFAT

The format from 2006 is a further development of the classic FAT formatting. exFAT was originally designed for removable media and is therefore particularly suitable for USB sticks, memory cards and external hard disks. It works especially effective with rather smaller data storage devices. Nevertheless, it can also handle large files and far exceed the FAT limit of 4 GB.

FAT32

The file system has existed since 1980. The variants released since then are known as FAT12, FAT16 and FAT32. FAT formatting is ideal for managing and exchanging small amounts of data. From today’s point of view, the FAT is obsolete because even in the current most powerful variant (FAT32 from 1997). Files can have a maximum size of 4 GB. Moreover, the maximum partition size is limited to 8 TB.

Despite these limitations, FAT formatting is still widely used. The main reason is that every OS can handle FAT32.
